# Web App Use Cases

Here’s a compiled list of all the **use cases** in a structured format:

---

### **Dictator Use Cases**

1. **Story Creation**
   - **As a**: User
   - **I want to**: Create a story with a name and a level
   - **So that**: I can add chapters to the story later.

2. **Chapter Creation**
   - **As a**: User
   - **I want to**: Create a chapter on a story by giving it a name, a transcript, and uploading its audio clips
   - **So that**: I can run sessions on the chapter.

3. **History Viewing**
   - **As a**: User
   - **I want to**: View my session history
   - **So that**: I can see my previous sessions and their details.

4. **Register**
   - **As a**: User
   - **I want to**: Register an account
   - **So that**: I can log in and use the system.

5. **User Login**
   - **As a**: User
   - **I want to**: Log in
   - **So that**: I can use the system.

6. **Schedule Chapter Session**
   - **As a**: User
   - **I want to**: After completing each session, schedule the next session for the current chapter
   - **So that**: I can maintain my learning schedule using spaced repetition.

7. **View Schedule**
   - **As a**: User
   - **I want to**: View my learning schedule
   - **So that**: I can see which chapters I should study and when.

8. **Update Schedule**
   - **As a**: User
   - **I want to**: Update the next scheduled date of a chapter in my schedule
   - **So that**: I can reschedule the next session for that chapter.

9. **Start a Session**
   - **As a**: User
   - **I want to**: Start a session for a particular chapter
   - **So that**: I can participate in a session for that chapter.
